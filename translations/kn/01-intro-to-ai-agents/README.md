# ಕೃತಕ ಬುದ್ಧಿಮತ್ತೆ ಏಜೆಂಟ್ಗಳು ಮತ್ತು ಏಜೆಂಟ್ ಬಳಕೆ ಪ್ರಕರಣಗಳಿಗೆ ಪರಿಚಯ

**ಸुरುಳಿದವರಿಗೆ ಕೃತಕ ಬುದ್ಧಿಮತ್ತೆ ಏಜೆಂಟ್ಸ್** ಕೋರ್ಸ್‌ಗೆ ಸ್ವಾಗತ! ಈ ಕೋರ್ಸ್ ನಿಮಗೆ ಮೂಲಭೂತ ಜ್ಞಾನವನ್ನು ಮತ್ತು ನಿಜವಾದ ಕೆಲಸ ಮಾಡುವ ಕೋಡ್ ಅನ್ನು ಒದಗಿಸುತ್ತದೆ — ಶೂನ್ಯದಿಂದ ಕೃತಕ ಬುದ್ಧಿಮತ್ತೆ ಏಜೆಂಟ್ಸ್ ನಿರ್ಮಿಸಲು.

<a href="https://discord.gg/kzRShWzttr" target="_blank">Azure AI Discord ಸಮುದಾಯ</a> ನಲ್ಲಿ ನಮಸ್ಕಾರ ಹೇಳಿ — ಇದು ಪ್ರಶ್ನೆಗಳಿಗೆ ಉತ್ತರ ನೀಡಲು ಸಂತೋಷಪಡುವ ಕಲಿಕಾರರು ಮತ್ತು AI ನಿರ್ಮಾಪಕರಿಂದ ತುಂಬಿದೆ.

ನಾವು ನಿರ್ಮಾಣಕ್ಕೆ ಮುಂಚೆ, ಕೃತಕ ಬುದ್ಧಿಮತ್ತೆ ಏಜೆಂಟ್ *ಎನು* ಎಂಬುದನ್ನು ಹಾಗೂ ಯಾವಾಗ ಅದನ್ನು ಬಳಸುವುದು ನ್ಯಾಯಸಮ್ಮತ ಎಂಬುದನ್ನು ನಿಜವಾಗಿಯೂ ಅರ್ಥಮಾಡಿಕೊಳ್ಳುವುದು ಅವಶ್ಯಕ.

## ಪರಿಚಯ

ಈ ಪಾಠದ ಒಳಗೊಂಡವು:

- ಏನು ಕೃತಕ ಬುದ್ಧಿಮತ್ತೆ ಏಜೆಂಟ್ಸ್ ಮತ್ತು ಅವುಗಳ ವಿಭಿನ್ನ ಪ್ರಕಾರಗಳು
- ಯಾವ ರೀತಿಯ ಕಾರ್ಯಗಳಿಗೆ ಕೃತಕ ಬುದ್ಧಿಮತ್ತೆ ಏಜೆಂಟ್ಸ್ ಅತ್ಯುತ್ತಮವಾಗಿ ಹೊಂದಿಕೊಳ್ಳುತ್ತಾರೆ
- ಏಜೆಂಟಿಕ್ ಪರಿಹಾರ ವಿನ್ಯಾಸದ ಸಮಯದಲ್ಲಿ ಬಳಸುವ ಪ್ರಮುಖ ಕಟ್ಟಡ ಬ್ಲಾಕ್ಗಳು

## ಕಲಿಕಾ ಗುರಿಗಳು

ಈ ಪಾಠದ ಅಂತ್ಯಕ್ಕೆ ನೀವು ಮಾಡಬಲ್ಲಿರಿ:

- ಏನು ಕೃತಕ ಬುದ್ಧಿಮತ್ತೆ ಏಜೆಂಟ್ ಮತ್ತು ಸಾಮಾನ್ಯ AI ಪರಿಹಾರದಿಂದ ಅದು ಹೇಗೆ ಭಿನ್ನವಾಗುತ್ತದೆ ಎಂಬುದನ್ನು ವಿವರಿಸುವುದು
- ಯಾವಾಗ ಕೃತಕ ಬುದ್ಧಿಮತ್ತೆ ಏಜೆಂಟ್ ಬಳಕೆ ಮಾಡಬೇಕು (ಮತ್ತೆ ಯಾವಾಗ ಅಲ್ಲ)
- ನಿಜವಾದ ಸಮಸ್ಯೆಗಾಗಿ ಒಂದು ಮೂಲಭೂತ ಏಜೆಂಟಿಕ್ ಪರಿಹಾರ ವಿನ್ಯಾಸವನ್ನು ರೇಖಾಚಿತ್ರ ರೂಪದಲ್ಲಿ ಬಿಡಿಸುವುದು

## ಕೃತಕ ಬುದ್ಧಿಮತ್ತೆ ಏಜೆಂಟ್ಸ್ ವ್ಯಾಖ್ಯಾನ ಮತ್ತು ಕೃತಕ ಬುದ್ಧಿಮತ್ತೆ ಏಜೆಂಟ್ಗಳ ಪ್ರಕಾರಗಳು

### ಏನು ಕೃತಕ ಬುದ್ಧಿಮತ್ತೆ ಏಜೆಂಟ್ಸ್?

ಇದನ್ನು ಸರಳವಾಗಿ ভাবಿಸೋಣ:

> **ಕೃತಕ ಬುದ್ಧಿಮತ್ತೆ ಏಜೆಂಟ್ಸ್ ಅಂದರೆ ದೊಡ್ಡ ಭಾಷಾ ಮಾದರಿಗಳ (LLMs) ನಿಜವಾಗಿಯೂ ಕಾರ್ಯ ನಿರ್ವಹಿಸಲು ವ್ಯವಸ್ಥೆಗಳು — ಪ್ರಾಂಪ್ಟ್ಗಳಿಗೆ ಪ್ರತಿಕ್ರಿಯಿಸುವುದಕ್ಕಿಂತಲೇ ಅವರು ಕೆಲಸ ಮಾಡಲು ಸಲಕರಣಗಳು ಮತ್ತು ಜ್ಞಾನವನ್ನು ಕೊಡುತ್ತದೆ.**

ನಾವು ಇದನ್ನು ಸ್ವಲ್ಪ ವಿಶ್ಲೇಷಿಸೋಣ:

- **ವ್ಯವಸ್ಥೆ** — ಒಂದು ಏಜೆಂಟ್ ಒಂದು ವಸ್ತು ಮಾತ್ರ ಅಲ್ಲ. ಅದು ಭಾಗಗಳ ಸಂಗ್ರಹವಾಗಿದ್ದು ಸೇರಿ ಕೆಲಸ ಮಾಡುತ್ತದೆ. ಮೂಲತಃ, ಪ್ರತಿಯೊಂದು ಏಜೆಂಟ್‌ಗೆ ಮೂರು ಅಂಶಗಳಿವೆ:
  - **ಪರಿಸರ** — ಏಜೆಂಟ್ ಕೆಲಸ ಮಾಡುವ ಸ್ಥಳ. ಉದಾಹರಣೆಗೆ, ಒಂದು ಪ್ರಯಾಣ ಬುಕ್ಕಿಂಗ್ ಏಜೆಂಟ್ ಗೆ ಬುಕ್ಕಿಂಗ್ ಪ್ಲಾಟ್‌ಫಾರ್ಮ್ ಆಗಿದೆ.
  - **ಸೇರನ್ಸ್** — ಏಜೆಂಟ್ ತನ್ನ ಪರಿಸರದ ಪ್ರಸ್ತುತ ಸ್ಥಿತಿಯನ್ನು ಓದುವ ವಿಧಾನ. ನಮ್ಮ ಪ್ರಯಾಣ ಏಜೆಂಟ್ ಹೋಟೆಲ್ ಲಭ್ಯತೆ ಅಥವಾ ವಿಮಾನ ದರಗಳನ್ನು ಪರಿಶೀಲಿಸಬಹುದು.
  - **ಚಟುವಟಿಕೆಗಳು** — ಏಜೆಂಟ್ ಕ್ರಮ ಕೈಗೊಳ್ಳುವ ವಿಧಾನ. ಪ್ರಯಾಣ ಏಜೆಂಟ್ ರೂಂ ಬುಕ್ ಮಾಡಬಹುದು, ದೃಢೀಕರಣ ಕಳುಹಿಸಬಹುದು, ಅಥವಾ ರದ್ದುಪಡಿಸಬಹುದು.

![ಕೃತಕ ಬುದ್ಧಿಮತ್ತೆ ಏಜೆಂಟ್ಸ್ ಎಂದರೆ ಏನು?](../../../translated_images/kn/what-are-ai-agents.1ec8c4d548af601a.webp)

- **ದೊಡ್ಡ ಭಾಷಾ ಮಾದರಿಗಳು** — ಏಜೆಂಟ್ಗಳು LLM ಗಳಿಗೆ ಮುಂಚೆ ಇದ್ದವು, ಆದರೆ LLM ಗಳು ಅಪೂರ್ವ ಸಾಮರ್ಥ್ಯವನ್ನು ನೀಡಿವೆ. ಅವು ಸಹಜ ಭಾಷೆಯನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳಬಹುದು, ಸನ್ನಿವೇಶವನ್ನು ವಿವೇಚಿಸಬಹುದು ಮತ್ತು ಬಳಕೆದಾರ ವಿನಂತಿಯನ್ನು ಸ್ಪಷ್ಟವಾದ ಕಾರ್ಯ ಯೋಜನೆಗೆ ಪರಿವর্তಿಸುವುದಕ್ಕೆ ಸಾಧ್ಯವಾಗುತ್ತದೆ.

- **ಕ್ರಮಗಳನ್ನು ನಿರ್ವಹಣೆ** — ಏಜೆಂಟ್ ವ್ಯವಸ್ಥೆಯಿಲ್ಲದೆ LLM ಸರಳವಾಗಿ ಪಠ್ಯಗಳನ್ನು ಉತ್ಪಾದಿಸುತ್ತದೆ. ಏಜೆಂಟ್ ವ್ಯವಸ್ಥೆಯೊಳಗೆ, LLM ನಿಜವಾಗಿಯೂ ಹಂತಗಳನ್ನು ನಿರ್ವಹಿಸಬಹುದು — ಡೇಟಾಬೇಸ್ ಹುಡುಕಾಟ, API ಕರೆ, ಸಂದೇಶ ಕಳುಹಿಸುವುದು.

- **ಸಲಕರಣಗಳ ಪ್ರವೇಶ** — ಏಜೆಂಟ್ ಬಳಸಬಲ್ಲ ಸಲಕರಣಗಳು (1) ಅದು ಕಾರ್ಯನಿರ್ವಹಿಸುವ ಪರಿಸರದಲ್ಲಿ ಮತ್ತು (2) ಅಭಿವೃದ್ಧಿಪಡಿಸಿದವರು ಏನನ್ನು ನೀಡಿದ್ದಾರೋ ಅವಲಂಬಿತ. ಪ್ರಯಾಣ ಏಜೆಂಟ್ ವಿಮಾನಗಳನ್ನು ಹುಡುಕಬಹುದು ಆದರೆ ಗ್ರಾಹಕ ದಾಖಲೆಗಳನ್ನು ಸಂಪಾದಿಸದು — ಇದು ನೀವು ಸಂಪರ್ಕಿಸುವುದರ ಮೇಲೆ.

- **ಸ್ಮರಣೆ + ಜ್ಞಾನ** — ಏಜೆಂಟ್ಗಳಿಗೆ ಚಿಕ್ಕಕಾಲದ ಸ್ಮರಣೆ (ಪ್ರಸ್ತುತ ಸಂಭಾಷಣೆ) ಮತ್ತು ದೈರ್ಘಕಾಲದ ಸ್ಮರಣೆ (ಗ್ರಾಹಕ ಡೇಟಾಬೇಸ್, ಹಳೆಯ ಸಂವಹನಗಳು) ಇರಬಹುದು. ಪ್ರಯಾಣ ಏಜೆಂಟ್ ನೀವು ಕಿಟಕಿಯ ಕುರ್ಚಿಗಳನ್ನು ಇಷ್ಟಪಡುತ್ತೀರಿ ಎಂದು "ಒತ್ತುಮಾಡಬಹುದು".

### ವಿಭಿನ್ನ ರೀತಿಯ ಕೃತಕ ಬುದ್ಧಿಮತ್ತೆ ಏಜೆಂಟ್ಗಳು

ಎಲ್ಲಾ ಏಜೆಂಟ್ಗಳು ಒಂದೇ ರೀತಿಯಲ್ಲಿ ನಿರ್ಮಿಸಲ್ಪಟ್ಟಿಲ್ಲ. ಈ ಕೆಳಗಿನ ಪಟ್ಟಿಯಲ್ಲಿ ಮುಖ್ಯ ಪ್ರಕಾರಗಳು ಮತ್ತು ಪ್ರಯಾಣ ಬುಕ್ಕಿಂಗ್ ಏಜೆಂಟ್ ನ ಉದಾಹರಣೆ:

| **ಏಜೆಂಟ್ ಪ್ರಕಾರ** | **ಏನು ಮಾಡುತ್ತದೆ** | **ಪ್ರಯಾಣ ಏಜೆಂಟ್ ಉದಾಹರಣೆ** |
|---|---|---|
| **ಸರಳ ಪ್ರತಿಕ್ರಿಯಾ ಏಜೆಂಟ್ಸ್** | ಕಠಿಣ-ಕೋಡ್ ಮಾಡಲಾದ ನಿಯಮಗಳನ್ನು ಅನುಸರಿಸುವುದು — ಸ್ಮರಣೆ ಇಲ್ಲ, ಯೋಜನೆಯಿಲ್ಲ. | ದೂರು ಇಮೇಲ್ ನೋಡುತ್ತದೆ → ಗ್ರಾಹಕ ಸೇವೆಗೆ ಕಳುಹಿಸುವುದು. ಅದೇಷ್ಟೇ. |
| **ಮಾದರಿ ಆಧಾರಿತ ಪ್ರತಿಕ್ರಿಯಾ ಏಜೆಂಟ್ಸ್** | ಜಗತ್ತಿನ ಆಂತರಿಕ ಮಾದರಿಯನ್ನು ಉಳಿಸುವುದು ಮತ್ತು ಬದಲಾವಣೆಗಳೊಂದಿಗೆ ನವೀಕರಿಸುವುದು. | ಹಳೆಯ ವಿಮಾನ ದರಗಳನ್ನು ಟ್ರ್ಯಾಕ್ ಮಾಡಿ, ಉನ್ನತ ದರಗಳ ಮಾರ್ಗಗಳನ್ನು ಗುರುತಿಸುವುದು. |
| **ಗೋಲ್ ಆಧಾರಿತ ಏಜೆಂಟ್ಸ್** | ಒಂದು ಗುರಿಯನ್ನು ಹೊಂದಿದ್ದು, ಅದನ್ನು ಹಂತ ಹಂತವಾಗಿ ತಲುಪುವ ಮಾರ್ಗವನ್ನು ಕಂಡುಹಿಡಿಯುವುದು. | ನಿಮ್ಮ ಪ್ರಸ್ತುತ ಸ್ಥಳದಿಂದ ಅಂತಿಮ ಗಮ್ಯಸ್ಥಾನಕ್ಕೆ ಪೂರ್ಣ ಯಾತ್ರೆಯನ್ನು ಬುಕ್ ಮಾಡುವುದು (ವಿಮಾನ, ಕಾರು, ಹೋಟೆಲ್). |
| **ಸೌಲಭ್ಯ ಆಧಾರಿತ ಏಜೆಂಟ್ಸ್** | ಏಕ ಪರಿಹಾರವನ್ನಲ್ಲ, ಉತ್ತಮ ಪರಿಹಾರವನ್ನು ತಟಸ್ಥತೆಗಳಿಗೆ ತೂಕ ನೀಡುವ ಮೂಲಕ ಹುಡುಕುತ್ತದೆ. | ವೆಚ್ಚ ಮತ್ತು ಅನುಕೂಲತೆ ನಡುವೆ ಸಮತೋಲನ ಮಾಡಿ ನಿಮ್ಮ ಪ್ರಾಧಾನ್ಯತೆಗಳಿಗೆ ಯಾತ್ರೆಯನ್ನು ಆರಿಸುವುದು. |
| **ಕಲಿಕಾ ಏಜೆಂಟ್ಸ್** | ಪ್ರತಿಕ್ರಿಯೆಯಿಂದ ಕಲಿಯುವ ಮೂಲಕ ಸಮಯಕ್ಕೆ ಉತ್ತಮವಾಗುತ್ತದೆ. | ಪ್ರಾಯಾಣ ನಂತರದ ಸಮೀಕ್ಷೆ ಪರಿಣಾಮಗಳ ಆಧಾರದ ಮೇಲೆ ಮುಂದಿನ ಬುಕ್ಕಿಂಗ್ ಶಿಫಾರಸ್ಸುಗಳನ್ನು ಸರಿಪಡಿಸುವುದು. |
| **ಅಧಿಕಾರಿತ ಏಜೆಂಟ್ಸ್** | ಉನ್ನತ ಮಟ್ಟದ ಏಜೆಂಟ್ ಕೆಲಸವನ್ನು ಉಪಕಾರ್ಯಗಳಿಗೆ ಹಂಚಿ ಕೆಳಮಟ್ಟದ ಏಜೆಂಟ್‌ಗಳಿಗೆ ವಿಂಗಡಿಸುತ್ತದೆ. | "ಯಾತ್ರೆ ರದ್ದುಮಾಡಿ" ವಿನಂತಿಯನ್ನು: ವಿಮಾನ ರದ್ದು, ಹೋಟೆಲ್ ರದ್ದು, ಕಾರು ರದ್ದು — ಪ್ರತಿಯೊಂದು ಉಪ ಏಜೆಂಟ್‌ಗಳಿಂದ ನಿರ್ವಹಣೆ. |
| **ಬಹು-ಏಜೆಂಟ್ ವ್ಯವಸ್ಥೆಗಳು (MAS)** | ಸ್ವತಂತ್ರ ಏಜೆಂಟ್‌ಗಳು ಒಟ್ಟಿಗೆ (ಅಥವಾ ಸ್ಪರ್ಧೆಯಲ್ಲಿ) ಕೆಲಸ ಮಾಡುತ್ತವೆ. | ಸಹಕಾರ: ವಿಭಿನ್ನ ಏಜೆಂಟ್‌ಗಳು ಹೋಟೆಲ್, ವಿಮಾನ, ಮನರಂಜನೆ ನಿರ್ವಹಣೆ. ಸ್ಪರ್ಧಾತ್ಮಕ: ಬಹು ಏಜೆಂಟ್‌ಗಳು ಉತ್ತಮ ಬೆಲೆಯಲ್ಲಿ ಹೋಟೆಲ್ ರೂомжನ್ನು ಭರ್ತಿಯಾಗಿಸಲು ಸ್ಪರ್ಧೆ. |

## ಎಪಿಗೆ ಯಾವಾಗ ಕೃತಕ ಬುದ್ಧಿಮತ್ತೆ ಏಜೆಂಟ್ರನ್ನು ಬಳಸಬೇಕು

ನೀವು *ಬಹುದಾದ* ಕಾರಣದಿಂದ ಏಜೆಂಟ್ ಬಳಕೆ ಮಾಡಬಹುದು ಎಂದರ್ಥ, ಯಾವಾಗಲೂ ಅದನ್ನು ಬಳಸಬೇಕು ಎಂಬುದಿಲ್ಲ. ಏಜೆಂಟ್‌ಗಳು ವಿಶೇಷವಾಗಿ ತೊಡಗಿಕೊಳ್ಳಲು ಸೂಕ್ತವಾದ ಪರಿಸ್ಥಿತಿಗಳು ಇಲ್ಲಿವೆ:

![ಕೃತಕ ಬುದ್ಧಿಮತ್ತೆ ಏಜೆಂಟ್ಸ್ ಯಾವಾಗ ಉಪಯೋಗಿಸಬೇಕು?](../../../translated_images/kn/when-to-use-ai-agents.54becb3bed74a479.webp)

- **ತೆರೆಯಲಾಗದ ಸಮಸ್ಯೆಗಳು** — ಸಮಸ್ಯೆಯನ್ನು ಪರಿಹರಿಸಲು ಹಂತಗಳನ್ನು ಪೂರ್ವನಿರ್ಧರಿಸಲಾಗದಾಗ. LLM dynamically ಮಾರ್ಗವನ್ನು ಕಂಡುಹಿಡಿಯಬೇಕು.
- **ಬಹು-ಹಂತ ಪ್ರಕ್ರಿಯೆಗಳು** — ಒಂದೇ ತಿರುವಿನಲ್ಲಿLookup ಅಥವಾ ಉತ್ಪಾದನೆ ಮಾತ್ರವಲ್ಲದೆ ಅನೇಕ ಹಂತಗಳಲ್ಲಿ ಸಲಕರಣಗಳನ್ನು ಬಳಸಬೇಕಾಗುವ ಕಾರ್ಯಗಳು.
- **ಕಾಲಕ್ರಮೇಣ ಸುಧಾರಣೆ** — ಬಳಕೆದಾರ ಪ್ರತಿಕ್ರಿಯೆ ಅಥವಾ ಪರಿಸರ ಸೂಚನೆಗಳ ಆಧಾರದ ಮೇಲೆ ವ್ಯವಸ್ಥೆ ಹೆಚ್ಚು ಶಕ್ತಿಯುಕ್ತವಾಗಬೇಕಾದಾಗ.

ಇದನ್ನು ಮುಂದಿನ **- ವಿಶ್ವಾಸಾರ್ಹ AI ಏಜೆಂಟ್ಸ್ ನಿರ್ಮಾಣ** ಪಾಠದಲ್ಲಿ ಇನ್ನಷ್ಟು ವಿವರವಾಗಿ ನೋಡಲಿದ್ದೇವೆ.

## ಏಜೆಂಟಿಕ್ ಪರಿಹಾರಗಳ ಮೂಲಭೂತಗಳು

### ಏಜೆಂಟ್ ಅಭಿವೃದ್ಧಿ

ಏಜೆಂಟ್ ನಿರ್ಮಿಸುವಾಗ ಪ್ರಥಮವು ನೀವು ಮಾಡಬೇಕಾದದ್ದು *ಅದು ಏನು ಮಾಡಬಹುದು* ಎಂಬುದನ್ನು ನಿರ್ದಿಷ್ಟಪಡಿಸುವುದು — ಅದರ ಸಲಕರಣಗಳು, ಕ್ರಮಗಳು, ಹಾಗೂ ವರ್ತನೆಗಳು.

ಈ ಕೋರ್ಸ್‌ನಲ್ಲಿ ನಾವು **Azure AI Agent Service** ಅನ್ನು ಪ್ರಮುಖ ವೇದಿಕೆಯಾಗಿ ಬಳಸುತ್ತೇವೆ. ಇದು ಬೆಂಬಲಿಸುವುದು:

- OpenAI, Mistral, Llama ಮೊದಲಾದ ತೆರೆದ ಮಾದರಿಗಳು
- Tripadvisor ಮೊದಲಾದ ಶ್ರೇಯಾಂಕಿತ ಡೇಟಾಪ್ರದಾಯಕರ ಲೈಸೆನ್ಸ್ಡ್ ಡೇಟಾ
- OpenAPI 3.0 ಸಲಕರಣ ವ್ಯಾಖ್ಯಾನಗಳ ಸ್ತರೀಕರಣ

### ಏಜೆಂಟಿಕ್ ಮಾದರಿಗಳು

ನೀವು LLM ಗಳೊಂದಿಗೆ ಪ್ರಾಂಪ್ಟ್ ಗಳ ಮೂಲಕ ಸಂವಹನ ಮಾಡುತ್ತೀರಿ. ಏಜೆಂಟ್‌ಗಳೊಂದಿಗೆ ಪ್ರತಿ ಪ್ರಾಂಪ್ಟ್ ಅನ್ನು ಕೈಯಿಂದ ರಚಿಸಲು ಸಾಧ್ಯವಿಲ್ಲ — ಏಜೆಂಟ್ ಅನೇಕ ಹಂತಗಳಲ್ಲಿ ಕ್ರಮ ತೆಗೆದುಕೊಳ್ಳುವುದು ಅಗತ್ಯ. ಅದಕ್ಕಾಗಿ **ಏಜೆಂಟಿಕ್ ಮಾದರಿಗಳು** ಇವೆ. ಅವು LLM ಗಳನ್ನು ಹೆಚ್ಚಿನ ಪ್ರಮಾಣದಲ್ಲಿ, ವಿಶ್ವಾಸಾರ್ಹವಾಗಿ ವಿನ್ಯಾಸಗೊಳಿಸಲು ಪುನಃಬಳಕೆ ಮಾಡಬಹುದಾದ ತಂತ್ರಗಳು.

ಈ ಕೋರ್ಸ್ ಅತ್ಯಂತ ಸಾಮಾನ್ಯ ಮತ್ತು ಉಪಯುಕ್ತ ಏಜೆಂಟಿಕ್ ಮಾದರಿಗಳ ಸುತ್ತಲೂ ರಚಿಸಲಾಗಿದೆ.

### ಏಜೆಂಟಿಕ್ ಫ್ರೇಮ್ವರ್ಕ್‌ಗಳು

ಏಜೆಂಟಿಕ್ Framework ಗಳು ವಿಕಸಕರಿಗೆ ಮಾಹಿತಿ, ಸಲಕರಣಗಳು ಮತ್ತು ಜಾಲಸಂಕೀರ್ಣಗಳನ್ನು ಪೂರ್ವನಿರ್ಮಿತ ಶೈಲಿಯಲ್ಲಿ ಒದಗಿಸುತ್ತವೆ. ಅವು ಸುಲಭಗೊಳಿಸುವುದು:

- ಸಲಕರಣಗಳ ಮತ್ತು ಸಾಮರ್ಥ್ಯಗಳ ಸಂಪರ್ಕ
- ಏಜೆಂಟ್ ಏಮಾಡುತ್ತಿದೆ ಎಂಬುದನ್ನು ಪರಿಶೀಲಿಸುವುದು (ಪರಿಭ್ರಮಣ ಕಾಲಿಕೊಳ್ಳಿ)
- ಬಹು ಏಜೆಂಟ್ ಗಳು ಸಹಕರಿಸುವುದು

ಈ ಕೋರ್ಸ್ ನಲ್ಲಿ ನಾವು **Microsoft Agent Framework (MAF)** ಮೇಲೆ ಗಮನ ಹರಿಸುತ್ತೇವೆ, ಇದು ಉತ್ಪಾದನಾ ಮಟ್ಟದ ಏಜೆಂಟ್‌ಗಳ ನಿರ್ಮಾಣಕ್ಕೆ ಸಹಾಯಕ.
